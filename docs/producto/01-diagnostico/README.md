# Diagnóstico

## ¿Para qué sirve esta fase?

El diagnóstico responde a la pregunta de Rumelt: **"¿Qué está pasando aquí?"**. Sin un buen diagnóstico, la política guía es ciega y las acciones son aleatorias.

## Orden del diagnóstico: de fuera hacia dentro

En 540 abordamos el diagnóstico en este orden:

1. **[Estudio de mercado](01-estudio-mercado.md)** — Mirar fuera primero: qué hay alrededor, tendencias, regulación, tamaño.
2. **[Contexto del producto/empresa](02-contexto-producto.md)** — Situar al cliente dentro de ese mercado.
3. **[Estudio de competencia](03-estudio-competencia/)** — Quién más está en este mercado y cómo compiten.
4. **[User research](04-user-research/)** — Quiénes son nuestros usuarios, qué necesitan, cómo se comportan.
5. **[Síntesis: problemas y oportunidades](05-problemas-oportunidades.md)** — Destilar todo en una lista priorizada.

## Investigación activa

En las fases 1 y 3 (mercado y competencia), la IA debe usar **WebSearch y WebFetch activamente** para recopilar datos verificables. Toda afirmación que venga de fuera lleva trazabilidad inline:

```markdown
El mercado español de [X] vale 3.200 M€ anuales [[Fuente: CNMC informe 2025](https://...)].
```

## Salida del diagnóstico

Al terminar el diagnóstico debes tener:

- ✅ Un mapa claro del mercado
- ✅ Una lista de competidores con deep dive
- ✅ Personas tipo bien definidas con frustraciones y comportamientos
- ✅ Customer journeys con pain points por fase
- ✅ JTBD (Jobs To Be Done)
- ✅ Una **lista de oportunidades** extraídas de personas + journeys + JTBD (alimenta el OST)
- ✅ Una **síntesis** con los 5-10 problemas core priorizados

Esto alimenta directamente las dos siguientes fases: **Opportunity Solution Tree** y **Política guía**.
